# M2 视角修正方案：俯角视图规范

**文档创建时间**: 2026-07-09  
**问题来源**: PM验收后用户反馈，当前背景和BOSS视角不符合俯角纵向卷轴要求  
**修正负责人**: Design Agent  
**验收标准**: 所有修正后的资产必须严格遵循iFighter 2风格的纯俯角视图

---

## 问题概述

《Flying Tigers 1945》是**俯角视图（Top-Down）纵向卷轴射击游戏**：
- 玩家飞机从**屏幕底部向顶部飞行**（由下往上）
- 玩家/敌机都在地面**上方**飞行
- 玩家看到的是**从空中向下俯视地面**的景象
- 类似 iFighter 2 / 1945 的经典俯角STG视角

### 错误现状

当前M2交付的24张背景图中，**远景层(far)全部错误**：
- 所有6关的 `*_far.png` 都是**侧视风景画**，包含大量天空区域
- 地平线清晰可见，构图是"平视远眺"而非"俯视地面"
- 例如：仰光关卡能看到佛塔的侧面轮廓和大片天空，这完全错误——我们在高空俯视地面，不应该看到这么多天空

### 正确参考：iFighter 2 风格

根据用户上传的iFighter 2参考截图，正确俯角应该是：

| 正确特征 | 描述 |
|----------|------|
| **观看方向** | 从飞机座舱位置（空中）向**正下方偏前**俯视地面 |
| **天空占比** | 远景层天空占比 ≤ 10%（仅画面最顶部可能有一线天空），90%以上区域都是地面/海洋 |
| **可见内容** | 道路、桥梁、建筑屋顶、农田、海岸线、海面纹理 —— 全部是**顶面** |
| **坦克战舰** | 只能看到顶部：坦克能看到炮塔顶盖和车身顶部，战舰能看到甲板和炮塔顶部，**完全看不到船体侧面** |
| **透视角度** | 约 **75°-85° 俯视**（接近但不完全是90°垂直），保留轻微前倾以便看到前方地形 |

## BOSS 视角规范总结

| BOSS类型 | 正确视角 | 错误 |
|----------|----------|------|
| **空中BOSS**（轰炸机、战斗机） | 俯角15-20度前倾，能看到机身顶部 ✅ | 无需修改 |
| **海上BOSS**（战舰、航母、要塞） | 纯俯角75-85度，**只能看到甲板/炮塔顶部，完全看不到船体侧面** ❌ → ✅ | 当前混合视角同时可见侧面，需要重绘 |
| **陆上BOSS**（坦克、要塞） | 纯俯角，**只能看到炮塔顶盖和车身顶部** ❌ → ✅ | 如果画了侧面需要修正 |

## 交付验证标准

PM验收修正后的资产时，检查以下项：

### 背景检查清单
- [ ] 所有 `*_far.png` 天空占比 ≤ 10%（仅最顶部可能有一线）
- [ ] 90%以上区域是地面/海洋/地形的**顶面**
- [ ] 能看到道路、建筑屋顶、河流都是俯视形态
- [ ] 没有地平线横穿画面中下部

### BOSS检查清单
- [ ] 战舰BOSS：只能看到甲板，看不到船体侧面
- [ ] 坦克/要塞BOSS：只能看到炮塔顶部，看不到车身侧面
- [ ] 所有BOSS长轴沿**纵向（屏幕上下方向）**排列，艏部朝上符合卷轴方向
- [ ] 保持512x512尺寸，RGBA透明背景

### 技术检查
- [ ] 所有文件PNG-32 RGBA格式（真正的PNG，不是JPG重命名）
- [ ] 文件名和目录结构完全不变（覆盖替换）
- [ ] 文件大小合理：单张背景 ≤ 2.5MB，单张BOSS ≤ 500KB
