# Flying Tigers 1945 — v1.5.0 升级设计文档

> **版本**: v1.5.0 Upgrade Plan v1.3
> **日期**: 2026-07-23
> **基准**: M3 完成状态（commit `d406392`）
> **参考**: claude GDD v1.0（2026-07-22）+ 原始 design-spec + M3-F 军衔系统
> **核心目标**: 关卡设计贴合历史原型、保留舰艇 BOSS 作为不可击沉的中场 BOSS、引入陆上大型单位、丰富 BOSS 多样性、**核心修正：背景采用纯俯视单层长条地图（非视差多图层）**、**v1.3 修正：隐藏情报改为关卡剧情事件（非随机掉落）**

## 1. 升级哲学

v1.5.0 的核心原则是 **"历史沉浸优先，街机爽感不减"**：

1. **关卡重排**：按 1941~1945 时间线重排 10 个主线关，每关对应真实战役
2. **舰艇 BOSS 保留但改机制**：保留妙高/天龙/最上等真实舰艇作为"不可击沉中场 BOSS"，通过 assault_phase（限时摧毁炮塔）胜利
3. **陆上大型单位 BOSS**：增加筑垒要塞/弹药库/机场塔台/碉堡群/油库群等陆上大型目标作为 BOSS
4. **保留精华**：深渊模式、军衔系统、隐藏情报、对象池等 M3 成果全部保留
5. **玩法多样化**：引入炸桥/对舰/战略轰炸/基地防卫/护送等不同关卡类型
6. **多战机系统**：从单架 P-40 扩展为 7 架可选战机
7. **BOSS 无变身**：取消二阶段纹理切换（phase1→phase2），改为弱点暴露/召唤护航/多目标等更真实的 BOSS 机制
8. **震电改放入 H2 广岛**：作为 1945 日本本土防空战实际参战过的局地战斗机，J7W 震电在 H2 广岛作为终极拦截 BOSS 出现

## 3. BOSS 重设计方案（v1.1 修订版）

### 3.0 v1.1 重大变更

经 PM 与用户讨论后，对 v1.0 草案进行了三方面重大调整：

1. **舰艇 BOSS 全部保留**（不再"去舰艇化"），但改为**不可击沉的中场 BOSS**（assault_phase 机制）
2. **增加陆上大型单位 BOSS**（筑垒要塞/弹药库/机场塔台/碉堡群/油库群）
3. **震电改 J7W2 保留**，作为 H2 广岛的终极拦截 BOSS

### 3.1 核心机制变更

#### 3.1.1 取消 phase 变身

**旧机制**（v1.0~v1.4）：
- BOSS 有 2~3 个阶段，每个阶段切换纹理（phase1.png → phase2.png）
- 阶段切换触发变身动画 + 弹幕变化

**新机制**（v1.5.0）：
- BOSS **无纹理切换变身**，改为 **动态弱点系统 + 时间线性难度曲线**
- BOSS 全程保持同一外观，但 **弱点区域周期性暴露/切换**
- 弹幕随时间推移逐步增强（difficulty_curve 线性插值）
- 编队类 BOSS 可逐架击落（如 Ki-48 三机编队，击落一架后剩余两架弹幕加密）

#### 3.1.2 新增 assault_phase 机制（不可击沉 BOSS）

针对舰艇和大型陆上设施（弹药库、机场塔台）：

- BOSS 主体不可击沉/击毁，HP 设为 `int.MAX` 或 `indestructible: true` 标志
- BOSS 包含多个 **部件节点**（炮塔、防空炮、雷达、弹药堆），每个部件独立 HP
- **胜利条件**：限定时间（如 60~90s）内摧毁所有部件
- **失败条件**：玩家在 assault_phase 内死亡 → Game Over
- **退场动画**：所有部件摧毁后 BOSS 冒烟/起火减速离场，弹出 "AC-130 撤退" 提示

#### 3.1.3 BOSS 类型分类

| 类型 | 描述 | 击破条件 | 关卡 |
|------|------|---------|------|
| `formation` | 敌机编队（可击毁） | 击毁领头机/全部击毁 | L01, L09 |
| `ace` | 王牌单机（可击毁） | 直接击毁 | L04（机场段）, H1 |
| `naval_assault` | **不可击沉舰艇** | 限定时间内摧毁所有部件 | L02, L04（海峡段）, L08 |
| `ground_facility` | 地面大型设施 | 击毁主体 / 摧毁所有部件 | L03, L05, L06, L10 |
| `multi_target` | **多目标群**（编队/船队/碉堡群） | 依次击破 | L07, L10 |
| `mixed` | 空中+地面混合 | 分别击破 | L08, L10 |
| `final` | 终极 BOSS | 击毁主体 | H2 |
| `environmental` | 环境 BOSS | 击毁 + 环境配合 | H1 |

## 15. 地图设计（v1.2 核心修正）

### 15.1 关键原则修正（重要！）

#### 15.1.1 错误确认

**M1~M3 阶段错误**：基于"横向卷轴"思维，错误采用了 4 层视差结构（far/mid/near/ground），共交付 65+ 张 PNG。这些图片含有"地平线/天际线/远山"等元素，**完全不符合纵向卷轴 STG 的纯俯视特性**。

**正确理解**：纵向卷轴 STG 是 **从正上方 90° 俯视地面**。玩家视野是"地面像地图一样"从下往上滑过：
- 看不到天空
- 看不到远处地平线
- 看不到天际线
- 看不到远处云朵
- **完全看到的是地面纹理**（土地、海洋、河流、山脉的俯视图）

#### 15.1.2 正确架构

```
纵向卷轴 STG 正确架构（v1.2）：

屏幕（1080x1920）         玩家飞机
┌────────────────┐              ▲
│  起跑线 Y=2400 │  ← 起点      │
│  城市/机场     │              │
│                │              │
│  中段 Y=1200   │              │
│  战场/河流     │   ↑↑↑        │   ↑↑↑
│                │   地图滚动    │   飞机
│  终点 Y=0      │   方向        │
│  终点/目标     │              │
└────────────────┘

单图层长条地图规格：
- 800x2400 像素
- 单张 PNG
- 90° 纯俯视
- 包含全部地面对象（道路、建筑、田地、河流、桥梁、山脉的俯视图）
- 滚动速度：16 px/s（关卡 150s）
```

#### 15.1.3 历史参考

iFighter 2 太平洋战争 1945（用户最常引用的参考）：
- 整张地图就是一张从上到下的卫星图
- 机场就是地面上的"跑道+机库"俯视
- 海洋就是深蓝色平面（可能有波纹但无地平线）
- 山脉是地图上的"棕色斑块"（不可飞入）
- 玩家从底部飞到顶部，整个地图匀速下移

### 15.2 通用地图规格

| 项目 | 规格 |
|------|------|
| 画布尺寸 | **800×2400** 像素（单图层长条） |
| 视角 | **90° 纯俯视**，零透视 |
| 天空 | **不存在**（无云、无天际线、无地平线） |
| 比例尺 | 1px ≈ 5m（2400px = 12km 实际距离） |

## 18. v1.3 隐藏情报系统重新设计

### 18.1 设计哲学修正

**v1.1 错误**：把情报做成"精英怪随机掉落"（本质是刷怪奖励），完全失去"隐藏要素"的意义。

**v1.3 正确设计**：每个情报都是**关卡内剧情事件**，具备四要素：
1. **前置条件**：玩家必须先在限定时间内完成特定动作（打怪/发现/等待）
2. **可视化提示**：地图上有位置标记，但需要玩家"主动发现"
3. **紧迫性**：目标出现后必须在限定时间内击毁，否则错过机会
4. **伴随威胁**：目标逃窜时伴随其他敌机/弹幕掩护，干扰玩家追击
5. **一次机会**：错过本关则本关永久无法获得（需重玩关卡）

### 18.2 通用机制规范

| 机制 | 实现方式 | 玩家感受 |
|------|---------|---------|
| 限时击毁小怪 | EventManager 计时器，8~12 秒 | 紧张感 |
| 接近发现动作 | 玩家飞机 50px 范围内触发 | 探索感 |
| 探照灯发现 | 建筑顶部有闪烁光源 | 视觉提示 |
| 多次选择 | 一关可能有多条情报线 | 决策感 |

| 紧迫性 | 提示方式 | 玩家后果 |
|--------|---------|---------|
| 倒计时显示 | 屏幕右上角红色数字 | 视觉压力 |
| 屏幕抖动 | 临近倒计时屏幕边缘抖动 | 触觉反馈 |
| 退出边界检测 | 目标飞出屏幕判定 | 即时反馈 |
| 永久提示 | 失败后弹"情报已转移" | 明确结果 |

| 伴随威胁 | 数量 | 出现时机 | 玩家影响 |
|---------|------|---------|---------|
| 护航战机 | 2~3 架 | 情报目标出现后立刻 | 强制分心 |
| 后续增援 | 4~6 架 | 目标逃窜中陆续 | 持续压力 |
| 场景干扰 | 1~2 个 | 关键路径上 | 阻断追击路线 |
| 弹幕强化 | 难度曲线提升 | 情报战斗期间 | 全屏压力 |

| 失败与重玩 | 规则 |
|-----------|------|
| 错过情报后本关永久错过 | 存档标记 `intel_missed_<stage_id>` |
| 玩家可重玩本关再次挑战 | 重玩不影响主线通关 |
| 不影响主线通关 | 仅影响 H1/H2 解锁进度 |
