## 1. 近期设计讨论演进

### 1.1 整体节奏

```
v1.0  (2026-07-23)  初版设计文档，关卡重排 + BOSS 去舰艇化
v1.1  (2026-07-23)  舰艇 BOSS 全部保留（assault_phase 机制），增加陆上大型单位 BOSS
v1.2  (2026-07-23)  核心修正：背景 4 层视差 → 单层长条纯俯视
v1.3  (2026-07-23)  核心修正：隐藏情报从"精英怪随机掉落"改为"关卡剧情事件"
v1.4  (2026-07-23)  二轮修正：① 情报简化为"打就完事" ② 4 个情报重新分布到 L02/L04/L05/L07
v1.5  (2026-07-23)  三轮修正：① 删除误伤扣分 ② 新增 4 关友军保护系统 ③ 统一奖励 +5000/关
```

### 1.2 关键决策回顾（v1.3 → v1.5）

| # | 议题 | 早期方案 | 最终方案（v1.5）| 决策点 |
|---|------|---------|----------------|--------|
| 1 | 隐藏情报形式 | 精英怪随机掉落 | 关卡剧情事件 | v1.3 |
| 2 | 情报收集机制 | 悬停/等待/探照灯等复杂交互 | 简单"打就完事" + 牛皮纸袋拾取 | v1.4 |
| 3 | 4 个情报位置 | L04 列车/L02 河中/L05 震电/L07 轿车 | **L02 地堡（驼峰）/ L04 轿车（东京）/ L05 军火库（衡阳战况）/ L07 运输舰（广岛）** | v1.4 |
| 4 | 误伤友军惩罚 | 扣 1000 分 + 取消 Combo | **完全删除**，友军按 NPC 处理 | v1.5 |
| 5 | 友军单位定位 | 不可攻击但可误伤 | **不可攻击**（子弹穿过），但可作为保护目标 | v1.5 |
| 6 | 友军保护情节 | 无 | **L03/L05/L07/L10 各 1 个**（+5000 分/关） | v1.5 |
| 7 | 失败惩罚 | 扣分/扣命 | **无惩罚**，仅弹"友军损失"提示 | v1.5 |
| 8 | 奖励力度 | 不确定 | **统一 +5000 分**（= 2-3 个精英怪分）| v1.5 |

### 1.3 v1.5 核心改动一览

**情报系统（4 个情报，分布在 4 个主线关）**：
- L02 仰光树林 → 驼峰航线通行证（击毁地堡型）
- L04 新竹机场 → 东京轰炸坐标（追击逃逸型，黑色轿车）
- L05 衡阳壕沟 → 衡阳战况密报（两段式，军火库+精英坦克）
- L07 湘江江面 → 广岛目标坐标（追击逃逸型，运输舰）

**友军保护系统（4 个保护情节，分布在 4 个主线关）**：
- L03 怒江 → 掩护远征军架桥（机枪阵地 ×3 + 95 式装甲车/Ki-27/Ki-48）
- L05 衡阳 → 死守城内据点（机枪阵地 ×4 + 97 式改坦克/Ki-45）
- L07 湘江 → 护航渡江运输船（运输船 ×2 + Ki-21/Ki-43）
- L10 芷江 → 保卫机场核心区（高炮阵地 ×4 + Ki-45/Ki-84）

**M3 保留系统**：深渊模式、军衔系统（7 级 PVT→ACE）、对象池、虚拟摇杆、存档系统、MapObjectManager — 全部保留

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| BOSS JSON 迁移工作量大 | M4-A 延期 | 优先迁移 L01-L05（高频关卡），L06-L10 用脚本自动生成 |
| 65+ 张旧背景清理耗时 | Design 部门压力大 | D14 列为 P1，先 P0 任务并行进行 |
| 友军保护系统事件复杂度 | C13 工作量超预期 | 4 关共用 `protect_ally_event` 类型，复用 80% 代码 |
| 牛皮纸袋道具跨场景拾取 | C12 与 EventManager 集成问题 | 牛皮纸袋作为 PowerUp 派生类，复用现有拾取逻辑 |
| 隐藏情报触发与关卡 BOSS 冲突 | 某些关卡情报和 BOSS 时间重叠 | 情报触发 Y 区域与 BOSS 区域 Y 至少间隔 300px |
| 多战机系统重构侵入性强 | C9 影响所有战机场景 | 引入适配器模式，旧战机场景无需改动 |
